Compiled lessons on DOM manipulation, forms, and attributes

Setting Attributes

You can use setAttribute() and removeAttribute() to modify element attributes directly in the DOM. 

const button = document.querySelector(".form__button");
button.setAttribute("type", "submit");

Boolean attributes: Setting a boolean like disabled only needs its presence to be true. 

button.setAttribute("disabled", true);
button.removeAttribute("disabled");

Attributes as properties: You can also access attributes through dot notation: 

const link = document.querySelector(".menu__link");
console.log(link.href);
link.href = "https://linkedin.com/";

Form Submission Events

Forms can submit data traditionally via HTML or through JavaScript event handling. 

const myForm = document.querySelector("#myForm");
myForm.addEventListener("submit", function (evt) {
evt.preventDefault();
// handle submission
});

Why use evt.preventDefault()? It stops the page from reloading when the form is submitted, allowing you to handle data dynamically.

Form Field Values

Most input fields store their data in the value property. You can access it directly using dot notation. 

const nameInput = document.querySelector(".form__input");
console.log(nameInput.value);

After submission, reset form fields with:

myForm.reset();

innerHTML and textContent

Use textContent for changing text, and innerHTML when you need to insert structured HTML. 

const title = document.querySelector(".article__title");
title.textContent = "Are NFTs the new way to collect art?";

⚠️ The Big Warning — Security

innerHTML is powerful, but dangerous. Using it with user-provided input can lead to Cross-Site Scripting (XSS) attacks.

  • ✅ Safe: when using your own trusted HTML.
  • 🚫 Unsafe: when using user-provided data.

📜 Summary

Property What It Does When to Use Danger
textContent Reads/sets plain text inside an element Use for simple text Overwrites all children
innerHTML Reads/sets HTML (tags + text) inside an element Use for structured HTML Security risk with user input
